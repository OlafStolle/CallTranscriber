### Task 2: Backend CORS für Produktion

**Files:**
- Modify: `backend/app/config.py`

- [ ] **Step 1: CORS Origins aus Env**

`config.py` hat bereits `allowed_origins: list[str] = ["http://localhost:3000"]`. Das reicht — wird über `.env` überschrieben mit `ALLOWED_ORIGINS=["https://transcriber.aicraftors.com"]`.

Prüfen: `allowed_origins` akzeptiert JSON-Liste in Env? Pydantic-Settings parsed `list[str]` als komma-separiert oder JSON.

Anpassung nötig — Pydantic-Settings parsed `list[str]` aus Env als JSON-String:

```python
from pydantic_settings import BaseSettings


class Settings(BaseSettings):
    supabase_url: str
    supabase_service_key: str
    supabase_jwt_secret: str
    openai_api_key: str
    upload_max_size_mb: int = 100
    allowed_origins: list[str] = ["http://localhost:3000"]
    model_config = {"env_file": ".env"}
```

Env-Format: `ALLOWED_ORIGINS=["https://transcriber.aicraftors.com","http://localhost:3000"]`

Kein Code-Change nötig — Pydantic-Settings v2 parsed JSON-Arrays aus Env nativ. ✅

### Task 4: Deploy-Script für VPS

**Files:**
- Create: `scripts/deploy.sh`

- [ ] **Step 1: Deploy-Script**

```bash
#!/bin/bash
set -euo pipefail

VPS_HOST="vps"
REMOTE_DIR="/opt/call-transcriber"

echo "=== Deploy Call Transcriber ==="

# Sync code to VPS
echo "1/4 Syncing code..."
rsync -avz --exclude='.git' --exclude='node_modules' --exclude='__pycache__' \
  --exclude='.next' --exclude='android' --exclude='.env' \
  /mnt/volume/Projects/call-transcriber/ ${VPS_HOST}:${REMOTE_DIR}/

# Build and start containers
echo "2/4 Building containers..."
ssh ${VPS_HOST} "cd ${REMOTE_DIR} && docker compose build"

echo "3/4 Starting containers..."
ssh ${VPS_HOST} "cd ${REMOTE_DIR} && docker compose up -d"

echo "4/4 Checking health..."
sleep 5
ssh ${VPS_HOST} "docker compose -f ${REMOTE_DIR}/docker-compose.yml ps"

echo "=== Deploy complete ==="
echo "API: https://transcriber-api.aicraftors.com/health"
echo "Web: https://transcriber.aicraftors.com"
```

- [ ] **Step 2: Migrations-Script**

```bash
#!/bin/bash
set -euo pipefail

# Apply Supabase migrations via psql
# Requires: SUPABASE_DB_URL in env (postgres://...)

if [ -z "${SUPABASE_DB_URL:-}" ]; then
  echo "ERROR: Set SUPABASE_DB_URL first"
  echo "Format: postgres://postgres.[project-ref]:[password]@aws-0-eu-central-1.pooler.supabase.com:6543/postgres"
  exit 1
fi

MIGRATIONS_DIR="$(dirname "$0")/../supabase/migrations"

for f in "${MIGRATIONS_DIR}"/*.sql; do
  echo "Applying: $(basename "$f")"
  psql "${SUPABASE_DB_URL}" -f "$f"
done

echo "All migrations applied."
```
